# v0.5.1 — Stability & Security

**Date:** 2026-06-02

## Philosophy

v0.5.1 is a stabilization release focused on fixing bugs, hardening security, and
resolving architectural regressions introduced by the new Core-Plugins kernel in v0.5.0.
No new features — only fixes, hardening, and performance improvements.

## Critical Fixes

- Fix `TypeError` crash in PRAL `_perceive()` — `recall()`/`query()` calls now match plugin method signatures
- Fix `AttributeError` crash in `TaoAgentCompat` — `--new-kernel` now works with `--state`, `--log`, `--dialogue`, and Ctrl+C handler
- Fix XSS vulnerability in Markdown renderer — `javascript:` and other dangerous URL schemes are now blocked in links and images

## Kernel & Plugin Fixes

- `Dispatch.call()` now surfaces handler errors instead of silently swallowing them — tool failures produce meaningful messages
- Drive system weights (observation/optimization/creation/maintenance) now correctly injected into system prompt in new kernel path
- Evaluation plugin dead code activated — `_collect_metrics()` stub no longer blocks snapshot generation
- Semantic store and skill catalog now use dirty-flag gating — write amplification eliminated on PRAL hot path

## MCP Protocol Compliance

- JSON-RPC batch requests now handled correctly (no longer crash the MCP server process)
- Positional (array) params now supported per JSON-RPC 2.0 spec (previously dropped silently)

## CLI & Launcher

- `tain run` now accepts a single agent name (matching `main.py` behavior) — help text corrected
- `tain` launcher `webui` subcommand auto-opens browser

## Internals

- Deprecated old Mixin files (`agent_config.py`, `agent_phase.py`, `agent_subsystems.py`, `agent_tools.py`, `agent_cognition.py`)
- Bumped default LLM model to MiniMax-M3
- `docs/report/` directory added to `.gitignore`

## Tests

- 328 tests (up from 326), 15 new test files added for kernel and plugin coverage
