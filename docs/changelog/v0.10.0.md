# v0.10.0 — Autonomous Evolution · Architecture Migration

**Date:** 2026-06-27

## Philosophy

v0.10.0 is a watershed release that closes the two biggest gaps identified in the
[insight report](../report/insight-report-2026-06-26.md): the evolution loop's
code-generation bottleneck and the Mixin-based architecture's structural debt.

The framework's core differentiation — "honest evolution" — is no longer an aspiration.
Agents can now autonomously complete the full gap→deploy→verify cycle with sandbox +
behavior contract + automatic rollback safety. The old `TaoAgent` + 6 Mixin architecture
has been replaced by a clean `AgentKernel` + 8 Plugin system with explicit interface
contracts.

## Architecture Migration: Mixin → Kernel/Plugin

The largest structural change in the project's history — ~2,788 lines deleted,
~2,600 lines of new architecture:

- **AgentKernel** (`kernel/`) — sole entry point replacing `TaoAgent`. Composed of
  `PRALLoop` (cognitive cycle), `LifecycleManager` (plugin lifecycle), and `Dispatch`
  (event routing)
- **PluginProtocol** (`kernel/protocol.py`) — explicit `@runtime_checkable` Protocol
  replacing 60+ `hasattr()` checks. Required: `initialize`, `shutdown`, `health_check`,
  `snapshot`, `restore`. Optional: `on_cycle_start`, `on_cycle_end`, `enrich_prompt`,
  `on_llm_response`
- **8 Plugins** — Identity, Memory, Tool, Skill, Knowledge, Workflow, Collaboration,
  Evaluation — each implementing PluginProtocol with clear boundaries
- **Deleted** — `agent.py` (723 lines), `bootstrap.py` (1,043 lines), `compat.py`,
  6 Mixin files (`agent_config`, `agent_subsystems`, `agent_cognition`, `agent_phase`,
  `agent_tools`, `agent_protocols`)
- **All consumers migrated** — `main.py`, `webui/`, `acp/server.py`, `dialogue.py`,
  `chat.py` now use `AgentKernel` + Plugin queries directly

## Autonomous Evolution Loop (v0.10.0)

The evolution loop is now closed — the agent can autonomously go from gap detection
to deployed tool with zero human intervention:

- **8-stage cycle** — GAP_DETECT → SPEC_DESIGN → CODE_GENERATE → CONTRACT_CHECK →
  SANDBOX_FORGE → REGISTER → ONLINE_VERIFY → EVALUATE
- **BehaviorContract** (`evolution/behavior_contract.py`) — every LLM-generated tool
  must declare its input/output schema, side effects, and max runtime. AST-based
  compliance verification before the code enters the sandbox
- **Three-layer safety** — sandbox (7-stage AST allowlist/blacklist) → behavior
  contract (import-vs-declaration verification) → automatic rollback (consecutive
  failure or quality degradation)
- **LLM code generation** — uses the agent's own LLM backend with sandbox allowlist
  injection into the prompt, eliminating the "human provides code" bottleneck
- **Automatic rollback** — tools that fail 3 consecutive online calls or degrade
  quality metrics are automatically rolled back to the previous known-good version
- **Background thread** — runs independently of the PRAL cognitive loop with rate
  limiting, quota enforcement, and state persistence across sessions

## v0.9.0 Stabilization

The debt-clearance iteration recommended by the insight report:

- **Version unification** — all 10+ hardcoded version strings replaced with
  `from tain_agent import __version__`; `pyproject.toml` synchronized
- **Dead code removal** — `SELF_DEFINE` phase methods, `external_world`/`trial_scheduler`
  tool categories removed from registration
- **Token deduplication** — `estimate_tokens` consolidated from 4 independent definitions
  to a single source in `utils/token_utils.py`
- **Config wiring** — `config.yaml` exploration parameters now actually drive phase
  transitions instead of being ignored
- **Retry consolidation** — manual retry logic in agent.py replaced with unified
  `llm_retry_call` from `retry.py`

## Testing

- **766 tests** — up from 614 in v0.5.2 (+152, 25% increase)
- **Evolution system tests** — pipeline, lineage, and emergence verifier now have
  dedicated test coverage (561 lines across 3 test files)
- **LLM stream parser tests** — XML tool call extraction, thinking block isolation,
  and edge cases (malformed XML, truncation, CDATA) fully covered
- **Autonomous evolution tests** — 41 tests covering BehaviorContract compliance,
  full 8-stage cycle, gap detection, contract enforcement, and rollback scenarios
- **Personality persistence tests** — cross-session disk fallback, agent-scoped
  workspace paths, T2→T1 sync

## Breaking Changes

- `TaoAgent`, `TaoAgentCompat`, and all Mixin classes removed — use `AgentKernel`
  with `STANDARD_FACTORIES` instead
- `connect_improvement_loop()` renamed to `connect_evolution_loop()` in `CognitiveLoop`
- `ImprovementLoop` replaced by `AutonomousEvolutionLoop` — the old `execute_once_if_needed()`
  API is preserved for backward compatibility
- `MetricsCollector` now accepts `evolution_loop` parameter (old `improvement_loop`
  still accepted as deprecated alias)
- Config schema now requires `evolution:` section with 5 fields (defaults provided
  in `config.yaml`)

## Internals

- `plugins/tool/` — `list_forged()`, `get_sandbox_allowlist()`, forge `action` param
- `plugins/knowledge/` — `GoalManager` with JSON persistence, `abandon()` support
- `plugins/identity/` — `_PersonalityAdapter` with `get_context_for_prompt()` and
  `introspect()`
- `plugins/memory/` — `session_memory` property for dialogue compatibility
- `kernel/prompts.py` — system prompts migrated from deleted `bootstrap.py`
- `kernel/factories.py` — `STANDARD_FACTORIES` dict with 7 Plugin factory mappings
- `core/config_schema.py` — `EvolutionConfigSchema` Pydantic model with field validators
- `config.yaml` — new `evolution:` section with defaults for all 5 config fields
- Web UI `agent_cache.py` — mtime-based `AgentKernel` caching replacing TaoAgent rebuild
- ACP `server.py` — session-scoped `AgentKernel` caching per ACP client
- `_TOOL_CATEGORY_MAP` — stale `external_fetch`/`external_subscribe`/`trial_status` entries removed
