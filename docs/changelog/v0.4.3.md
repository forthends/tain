# v0.4.3 — Stability, Observability & Boundary Expansion

**Date:** 2026-05-26
**Branch:** `release/v0.4.3`

## Highlights

- **LLM retry resilience** — exponential backoff with jitter for transient API failures, retryable vs non-retryable exception classification
- **Token-aware context** — tiktoken-based token estimation with auto-summarization and budget trimming to prevent context window overflow
- **Chat cancellation** — real-time SSE cancellation via `asyncio.Event`, dedicated API endpoint, and Send/Stop button toggle in Web UI
- **MCP integration** — dynamic external tool discovery from MCP servers over stdio (JSON-RPC)
- **ACP protocol** — JSON-RPC ACP server for embedding agents in external editors (Zed-compatible)
- **159 tests** across 10 test files covering all new modules

---

## P0: LLM Retry Mechanism (`tain_agent/core/retry.py`)

- `RetryConfig` — configurable exponential backoff with capped max delay and jitter
- `retry_call()` / `retry_stream()` — retry wrappers for sync call and streaming generators
- `_is_retryable()` — classifies exceptions by name and HTTP status code:
  - Retryable: connection errors, timeouts, 429 rate limits, 5xx server errors
  - Non-retryable: 400 bad request, 401 unauthorized, value/type errors
- `_calculate_delay()` — min(initial * base^attempt, max_delay) with ±10% random jitter
- Integrated into both `AnthropicBackend` and `OpenAICompatibleBackend`
- Config in `config.yaml` under `llm.retry`

## P1: Token-Aware Context Management (`tain_agent/core/conversation.py`)

- `estimate_tokens()` — tiktoken (cl100k_base) with character-based fallback (2 chars/token)
- `needs_summarization()` — checks against configured `token_limit`, skips on fresh histories
- `summarize()` — preserves first message + all user messages, compresses execution blocks between them using LLM summarization
- `trim_to_token_budget()` — removes middle messages while keeping first + last N
- `keep_first_and_last()` — boundary-safe trim that never breaks tool_use/tool_result pairs
- `ConversationManager` initialized with token config from `config.yaml`

## P2: Structured LLM Logging (`tain_agent/core/llm_logger.py`)

- JSONL-format logging to `agent_workspace/<name>/logs/llm_calls.jsonl`
- `log_request()` — captures provider, model, message count, estimated tokens, tool count
- `log_response()` — finish reason, content/thinking previews, token usage (prompt/completion), latency
- `log_tool_result()` — tool name, arguments, success/failure, result preview, latency
- All content previews truncated to configurable max lengths
- Integrated into `TaoAgent` initialization and `process_chat_message()` in web dialogue

## P3: Web Chat Cancellation (`webui/dialogue.py`, `webui/routes/api_chat.py`)

- Module-level `_active_cancel_events: dict[str, asyncio.Event]` registry
- `cancel_chat_message(message_id)` — signals cancellation for an active SSE stream
- `POST /api/agent/{name}/chat/cancel` — dedicated cancel API endpoint
- `_cleanup_incomplete_messages()` — removes orphaned tool_use blocks to prevent API errors
- Cancel event checked at turn boundaries and before summary turn in `process_chat_message()`
- Web UI: Send button toggles to red Stop button during streaming; `[已取消]` marker appended on cancel
- `message_id` generated per request and passed through SSE events

## P4: Agent Memory Tools (`tain_agent/tools/primal.py`)

- `remember_note(category, content)` — appends structured JSONL note to `agent_workspace/<name>/memory/agent_notes.jsonl`
- `recall_notes(category, limit)` — returns notes sorted by recency (newest first), optionally filtered by category
- Categories: discovery, user_preference, pattern, reflection, idea, decision, milestone
- `_get_notes_path()` — resolves notes path within agent workspace, auto-creates memory directory
- Proper parameter schemas with required/optional flags

## P5: Tool Base Class (`tain_agent/tools/base.py`)

- `Tool` abstract base class with mandatory `name`, `description`, `parameters`, `execute()` interface
- `to_schema()` — Anthropic-format `{name, description, input_schema}` with normalized JSON Schema
- `to_openai_schema()` — OpenAI-format `{type: "function", function: {...}}` with sanitized types
- `_normalize_parameters()` — converts flat `{param: {type, description, required}}` to standard JSON Schema with `properties`/`required` arrays
- `_sanitize_json_schema()` — recursively fixes invalid JSON Schema types (`Any→string`, `List[...]→array`, `Dict[...]→object`)
- `register_on(registry)` — integration with `ToolRegistry`
- `__call__()` — callable protocol for tool instances
- `ToolRegistry.register_tool()` — new method accepting `Tool` instances, stores instance reference for schema access

## P6: Forge SKILL.md Export (`tain_agent/tools/forge.py`)

- `ToolForge.export_as_skill(name)` — exports a forged tool as a SKILL.md package
- Output structure: `forged_tools/skills/{name}/SKILL.md` with `scripts/`, `references/`, `assets/` directories
- `_build_skill_md()` — generates YAML frontmatter (name, description, version, parameters) + markdown body
- `_format_params_md()` — renders parameters as markdown table
- SKILL.md format compatible with agent skill loading systems

## P7: Forge Tool Templates (`tain_agent/tools/templates.py`)

- `resolve_path(workspace_dir, path, for_write)` — safe path resolution rejecting workspace escapes
- `truncate_output(text, max_tokens=32000)` — head+tail preservation with truncation marker
- `truncate_lines(text, max_lines=5000)` — line-based truncation with head+tail
- `run_shell(command, timeout=30.0, workspace_dir=None)` — subprocess execution with timeout, captures stdout/stderr
- `format_error(message, exception=None)` — standard `{success: false, error, error_type, traceback}` output
- `estimate_tokens(text)` — tiktoken with character-based fallback
- Reusable utilities for forged tools, preventing reimplementation of common patterns

## P8: MCP Integration (`tain_agent/tools/mcp_loader.py`)

- `MCPClient` — manages MCP server subprocess over stdio transport
  - `start()` / `stop()` — subprocess lifecycle with merged environment
  - `list_tools()` — `tools/list` JSON-RPC call
  - `call_tool()` — `tools/call` JSON-RPC call with structured result extraction
  - `_send_request()` — JSON-RPC 2.0 over stdin/stdout with error handling
- `MCPTool(Tool)` — wraps remote MCP tools as local `Tool` instances, proxies `execute()` via MCP client
- `load_mcp_config(config_path)` — reads mcp.json config file
- `discover_and_register(registry, mcp_config_path)` — loads config, connects servers, discovers tools, registers on `ToolRegistry`
- Config format: `{"mcpServers": {"server-name": {"command": "...", "args": [...], "env": {...}}}}`
- Fault-tolerant: one failing server doesn't block the agent

## P9: Background Process Management (`tain_agent/tools/background_manager.py`)

- `BackgroundProcess` dataclass — id, command, asyncio.subprocess.Process, started_at, output_buffer
- `BackgroundShellManager` — async subprocess pool with dedicated event loop thread
  - `start(command)` → process_id, enforces `MAX_PROCESSES=5` limit
  - `get_output(process_id, tail_lines=50)` — buffered stdout lines
  - `kill(process_id)` — SIGTERM → 3s grace → SIGKILL
  - `list_processes()` — all processes with status, exit codes, output counts
  - `wait(process_id, timeout=30.0)` — async wait with timeout
  - `kill_all()` — cleanup on agent stop
- Exposed as primal tools: `bg_start`, `bg_output`, `bg_kill`, `bg_list`, `bg_wait`
- Module-level singleton with `init_manager()` / `get_manager()`
- Safety: commands run in workspace directory, process limit enforced, auto-cleanup on stop

## P10: ACP Protocol Support (`tain_agent/acp/server.py`)

- `ACPServer` — JSON-RPC 2.0 server over stdin/stdout
- Session management: `initialize` → `newSession` → `prompt` → `cancel` / `closeSession`
- `initialize(protocol_version, capabilities)` → `{protocolVersion, serverInfo, capabilities}`
- `newSession(workspace_path)` → `{session_id, workspace_path}`
- `prompt(session_id, text)` → streaming ACP events, reuses `process_chat_message()` internally
- `cancel(session_id)` → `{cancelled: true/false}`, reuses P3's `asyncio.Event` mechanism
- `closeSession(session_id)` → cleanup and session removal
- `_convert_to_acp_event()` — translates SSE events to ACP format (`text_delta`, `tool_call`, `thinking`, `done`, `cancelled`)
- Entry point: `python -m tain_agent.acp`
- Compatible with Zed editor and other ACP clients

## P11: Multi-Level Config Search (`tain_agent/core/config.py`)

- `load_config(explicit_path=None)` — layered config loading with deep merge:
  1. `--config` CLI flag (highest priority)
  2. `./config.yaml` (project root)
  3. `~/.tain/config.yaml` (user-level global)
  4. Built-in package defaults (lowest priority)
- `load_agent_overrides(config, agent_name, workspace_dir)` — per-agent `agent_workspace/<name>/agent.yaml` overrides
- `deep_merge(base, override)` — recursive dict merge (lists and scalars replace, not merged)
- `get_config_files()` — debug helper listing all config sources with priorities
- Non-existent or corrupt config files don't crash — defaults are always returned
- `DEFAULT_CONFIG` includes all sections: framework, agent, agent_workspace, llm (with retry), conversation, tools, logging

## P12: Smart File Truncation (`tain_agent/utils/token_utils.py`)

- `estimate_tokens(text)` — tiktoken (cl100k_base) with character-based fallback
- `truncate_text_by_tokens(text, max_tokens=32000, head_ratio=0.5)` — head+tail preservation with customizable split ratio, natural breakpoint detection at sentence/paragraph boundaries
- `truncate_lines(text, max_lines=5000)` — line-based truncation with head+tail
- Integrated into `read_file` in primal.py — large files auto-truncated with clear markers showing original size and limit
- Integrated into `_build_system_prompt()` in web dialogue — tool descriptions truncated to stay within token budget
- Config in `config.yaml` under `tools.max_output_tokens` and `tools.max_output_lines`
