# v0.4.2 — Web UI & Multi-Provider LLM Support

**Date:** 2026-05-26
**Branch:** `release/v0.4.2`

## Highlights

- **Multi-provider LLM backend** — support for DeepSeek, MiniMax, OpenAI, and Anthropic via a unified backend abstraction with native function calling
- **Web chat with agents** — real-time SSE-streamed dialogue with tool-call visualization, thinking indicators, and collapsible execution cards
- **XML tool call resilience** — namespace-agnostic parser with regex fallback that strips hallucinated XML before it reaches the user
- **Content sanitization pipeline** — conversation history is cleaned on load and on save, preventing model contamination from prior XML-formatted responses

---

## LLM Backend (`tain_agent/core/llm.py`)

- Added `AnthropicBackend` — Anthropic-native streaming with thinking block support; also works with MiniMax via `/anthropic` endpoint
- Added `OpenAICompatibleBackend` — OpenAI / DeepSeek streaming with tool call accumulation by index
- JSON Schema sanitization for OpenAI tools: invalid types (`Any`, `List[...]`, `Dict[...]`) are mapped to valid JSON Schema types
- Fixed `content: null` in assistant messages with tool_calls — DeepSeek V4 Flash requires non-null content
- Reasoning content (`reasoning_content`) preserved across multi-turn tool-call conversations
- Factory function `create_backend()` selects backend from config provider + base_url

## Web Chat (`webui/dialogue.py`)

- Full SSE streaming dialogue with agents via `process_chat_message()`
- Per-turn text buffering: text tokens are accumulated, checked for XML tool calls, stripped, then yielded in display-friendly chunks
- Multi-turn tool execution loop: up to 5 turns, tools removed after 3 uses to force summarization
- Forced summary turn when no text response produced after tool calls
- Lifecycle events: `status: thinking | text`, `tool_start`, `tool_done`, `done`
- XML tool call extraction (`_extract_xml_tool_calls`):
  - Namespace-agnostic tag matching: `minimap:`, `||DSML||`, full-width pipe variants
  - XML parsing with `ElementTree`, falling back to regex extraction when XML is malformed
  - `_extract_tool_calls_regex_fallback()` for unparseable XML
- `_chunk_text()` — splits text at natural boundaries (punctuation, newlines) for streaming feel
- Conversation history loaded/saved with automatic XML stripping

## Web UI

- **Chat tab** (`agent_tabs/chat.html`):
  - Thinking indicator with animated bounce dots ("思考中...")
  - Collapsible tool-call cards showing tool name, input preview, and status (running / done)
  - Markdown rendering via `marked.js`
  - Streaming text display with cursor pulse
  - Empty state with agent avatar and status
- **Agent detail** (`agent_detail.html`):
  - Tab navigation pushes canonical `?tab=` URLs (no more fragment endpoint in address bar)
  - HTMX-powered tab content loading with `hx-get` on fragment endpoints
  - Sidebar agent list with active highlighting
  - Agent start/stop/restart controls with status refresh
- **Decision log** (`agent_tabs/decisions.html`):
  - Phase/type filtering with HTMX partial updates
  - "Load More" pagination with OOB summary updates
- **Knowledge viewer** (`agent_tabs/knowledge.html`):
  - File listing with click-to-render
  - Content rendering via `webui/render.py` with format detection
- **Live output** (`agent_tabs/live.html`):
  - SSE-streamed agent daemon log output
  - Auto-scroll with toggle

## Routes

- `GET /agent/{name}/tab/{tab}` — fragment HTML for each tab panel
- `GET /agent/{name}/decisions-list` — filtered/paginated decisions
- `GET /agent/{name}/knowledge/render` — file content rendering
- `POST /agent/{name}/chat` — SSE chat endpoint
- `GET /api/agent/{name}/chat/history` — conversation history JSON
- `POST /agent/{name}/controls/{start,stop,restart}` — daemon lifecycle
- `GET /sidebar/agents` — sidebar agent list (HTMX partial)
- `GET /stream/agent-output` — per-agent log tail SSE

## Storage & Safety

- `tain_agent/storage_registry.py` — semantic storage registry to organize workspace files by category, combating entropy from scattered tool output
- `tools/cleanup_workspaces.py` — workspace cleanup utility
- Runtime state (`tain_agent/state/`) removed from git tracking
- Protected paths in config include new core modules

## Fixes

- Sidebar JSON leakage: agent metadata no longer rendered as raw JSON in HTML
- Broken agent controls: start/stop buttons now trigger daemon lifecycle correctly
- Page nesting: tab clicks fetch fragment HTML (`/agent/{name}/tab/{tab}`) instead of full page, then push canonical URL to address bar
- Tab bar sync: active tab state preserved after HTMX content swaps
- Chat send button stuck: fixed undefined `chatApp` reference when chat tab not initially loaded
- DeepSeek V4 Flash `content: null` error: empty assistant content with tool_calls now sent as `""`
- XML tool call leakage: buffering + sanitization prevents XML from appearing in chat or persisting to history

## Config Changes

- `config.yaml` llm section updated with documented alternatives for DeepSeek, Anthropic, and MiniMax backends
- Provider switchable via `provider`, `api_key_env`, and `base_url` fields

---

## Migration Notes

- Conversation histories from v0.4.1 may contain XML-formatted tool calls that contaminate context. Run the cleanup script or clear `agent_workspace/*/logs/conversations/web_user.jsonl`.
- `config.yaml` llm section structure unchanged — existing configs compatible.
- Base URL for MiniMax must include `/anthropic` suffix to use the Anthropic-compatible backend.
