# v0.5.2 ‚Äî Agent Pain-Point Resolution

**Date:** 2026-06-26

## Philosophy

v0.5.2 addresses structural framework bottlenecks identified by long-running Agents (puzzle and sage)
through 58 evolution cycles of independent observation. The Agents discovered that certain framework design
decisions ‚Äî not bugs ‚Äî were creating evolution ceilings. This release closes those gaps across three
Agent pain-point chains: tool self-evolution, personality lifecycle, and metacognition infrastructure.

## Chain A: Tool Self-Evolution Loop

- `forge_tool` now accepts `action="update"` to re-register existing tools through the full
  sandbox‚Üícompile‚Üíexec‚Üíregister pipeline, resolving the per-name write-once constraint
- Sandbox allowlist unified into a shared `sandbox_allowlist.py` module ‚Äî single source of truth
  for forge_tool sandbox validation
- `get_sandbox_allowlist` tool exposed to Agents so they can query available imports before
  writing forge code
- Sandbox failures now include the allowlist in error reports, eliminating 3-4 rounds of
  trial-and-error discovery
- Metrics snapshot paths normalized between `EvolutionReporter` and `quality_gate` ‚Äî
  both now use the canonical `agent_workspace/{agent_name}/state/metrics_snapshots/` path

## Chain B: Personality Lifecycle Management

- Personality now accepts `workspace_path` parameter for agent-scoped disk persistence ‚Äî
  puzzle and sage no longer share the same `personality.json`
- `_load_from_disk` fallback when Memory is empty enables cross-session personality continuity
- `sync_runtime_to_disk` provides T2 (runtime introspect)‚ÜíT1 (disk) sync at session end
- `auto_observe` traits now tagged with `source="auto_emergent"` for correct sync filtering
- 5 new persistence tests added for workspace_path and disk fallback coverage

## Chain C: Metacognition & Collaboration Infrastructure

- `get_self_profile` lightweight introspection API ‚Äî aggregated action distribution, tool
  usage ranking, trait activity, and active goals from framework-internal sources, replacing
  Agent-built heavy self-observation tool chains
- `diagnose_framework` tool for structured Agent‚Üíframework diagnostic feedback ‚Äî results
  written to `agent_workspace/{agent_name}/diagnostics/` for developer review
- Startup unread message check notifies Agents of pending inter-agent messages at session start

## Internals

- `evolution_metrics.py` `_snapshots_dir` fixed to treat `base_dir` as complete target path
  (eliminating redundant `tain_agent/` path nesting)
- quality_gate S7 and S9 gates now use unified `_find_snapshot_dir` with 3-tier fallback
- Sandbox allowlist constants extracted from `test_forged_tool.py` to shared module
- Stale `_workspace_dir` test mock updated to patch `_find_snapshot_dir`

## Tests

- 614 tests (up from ~610), 5 new personality persistence tests
